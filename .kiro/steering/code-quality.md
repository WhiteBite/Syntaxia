# Качество кода - Правила

## Запрещено в коде

| Что                    | Почему                           |
| ---------------------- | -------------------------------- |
| `console.log()`        | Использовать logger              |
| `// TODO` без задачи   | Оформлять в TODO-refactoring.txt |
| `any` в TypeScript     | Использовать конкретные типы     |
| Магические числа       | Выносить в constants             |
| Закомментированный код | Удалять перед коммитом           |

## Перед коммитом

```bash
# Backend
go build ./...
go test ./...

# Frontend
npm run build
npm run test:run
```

## Лимиты размеров

| Элемент       | Лимит       |
| ------------- | ----------- |
| Файл          | < 500 строк |
| Функция       | < 50 строк  |
| Vue компонент | < 300 строк |

**Если превышено:** разбить на модули/composables/подкомпоненты.

## Git commits

```
feat: добавить экспорт в PDF
fix: исправить утечку памяти
refactor: разбить tool_executor
test: добавить тесты для git_tools
docs: обновить README
```

## Ветки

- `main` - стабильная версия
- `feature/*` - новый функционал
- `fix/*` - исправления
- `refactor/*` - рефакторинг

## Комментарии

**Нужны:** публичные API, сложная логика, неочевидные решения

**Не нужны:** очевидный код, getters/setters, простой CRUD
